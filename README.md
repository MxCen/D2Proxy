# D2Proxy - KCP 游戏代理

## 项目简介

D2Proxy 是一个基于 **KCP 协议**的游戏网络代理系统，用于优化游戏连接质量。通过将 TCP 流量转换为可靠的 UDP 传输，降低延迟、减少丢包，提升弱网环境下的游戏体验。

## 架构设计

```
┌─────────────┐        ┌─────────────┐        ┌─────────────┐
│  游戏客户端  │  TCP   │  D2Client   │  UDP   │  D2Server   │  TCP   │  游戏服务器  │
│             │ ──────>│  (KCP客户端) │ ──────>│  (KCP服务端) │ ──────>│             │
└─────────────┘        └─────────────┘        └─────────────┘        └─────────────┘
                           本地代理                 远程代理
```

**数据流向**：`游戏客户端 → TCP → D2Client → KCP/UDP → D2Server → TCP → 游戏服务器`

## 核心模块

| 模块 | 说明 |
|------|------|
| **D2Client** | 客户端 DLL，注入游戏进程，Hook Socket API，拦截 TCP 连接并转发至本地代理 |
| **D2Server** | 服务端 DLL，监听 UDP 端口，接收 KCP 数据包并转发至游戏服务器 |

## 技术特性

- **KCP 协议**：可靠 UDP 传输，支持快速重传、拥塞控制
- **透明代理**：通过 API Hook 实现对游戏无感知的流量劫持
- **多会话管理**：支持多客户端同时连接，自动会话清理
- **低延迟优化**：nodelay 模式 + 自适应更新间隔

## 编译环境

- Visual Studio 2022
- Windows SDK (v141_xp)
- C++ 标准

## 快速使用

1. 编译生成 `D2Client.dll` 和 `D2Server.dll`
2. 服务端：注入 D2Server 到游戏服务器进程
3. 客户端：注入 D2Client 到游戏客户端进程
4. 配置 `KCP_GS_IP` 和 `KCP_GS_PORT` 指向游戏服务器地址
